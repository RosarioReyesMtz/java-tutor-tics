Problema 5.1 - Suma de Matrices Esparcidas

Problema 5.1 (pág. 354)

Suma de Matrices Esparcidas

Una matriz esparcida (mayoría de ceros) no se almacena completa. Solo se guardan los valores distintos de cero, junto con su renglón y columna. El programa debe: a) Leer matrices A y B, y guardarlas en arreglos de registros (A1, B1). b) Obtener la matriz C = A + B, usando solo A1 y B1. c) Imprimir C.

Requisitos

  • Datos: Dos matrices A(M x N) y B(M x N), 1 < M < 50, 1 < N < 30.
  • Definir una estructura (Registro) para almacenar: REN, COL, VAL.
  • a) Llenar los arreglos A1 y B1 (Arreglos de Registros) solo con los valores distintos de 0.
  • b) Calcular C(M x N) = A + B. La suma debe hacerse "mezclando" (merge) los arreglos A1 y B1, similar a un merge sort.
  • c) Imprimir la matriz C resultante.

Notas rápidas

  • Este problema introduce Arreglos de Registros (en Java, ArrayList de una clase personalizada).
  • La lógica de suma (inciso b) es el algoritmo de "merge" (fusión) más complejo de este conjunto de problemas.
